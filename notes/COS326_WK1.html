COS 326 -- Week 1: Intro to OCaml

Why OCaml?

Small, orthogonal coe based on the lambda calculus.

Control is based on recursive functions instead of loops.

Supports first-class (functions are data values), lexically scoped (statically typed), high-order (functions as arguments and results) procedures.

Similar languages are Clojure, Scala, F#, and Haskell.

OCaml is strongly and statically typed (won't compile unless types are correct).

OCaml comes with compilers:

  • ocamlc is a fast bytecode compiler
  • ocamlopt optimizing, native code compiler
  • ocamlbuild nice wrapper that computes dependencies.

Also top-level shell:

  • ocaml at the prompt

OCaml files have the extension .ml

Expressions are computations (2 + 3).

Values (a subset of the expressions) are the results of computations (5).

Types describe collections of values and the computations that generate those values (int is a type), values of type int include (max_int, min_int, 0, 1, 2).

char is single quote, string is double quote.

** Unit is similar to void except it is just a type with exactly one uninteresting value.

xxxxxxxxxx
42 * (13 + 1) -->* 588

This means that the expression evaluates to the value 588 in 0 or more small steps.

OCaml will never allow an expression of the type 1 + "hello", unlike JavaScript.

Write function expressions as f x y z this is the function f which takes three arguments, applied to the inputs x,y,z in that order.

Every value has a type and so does every expression

(e:t) means expression e has type t.

!! Programs that do not follow the rules will not type check and OCaml will refuse to compile them for you. This prevents runtime errors and allows us to better reason about programs.

There exist formal proofs that expressions are well-typed.

You can use the OCaml interpreter to find out the type of a simple expression.

Runtime errors still do exist, they are just less likely than, say, Python.

OCaml is a functional programming language (express control flow and iteration by defining functions).

OCaml is a typed programming language.

Simple Functions

A function is defined with the let keyword, optionally rec, a name, its parameters, its return type, and its expressions.

xxxxxxxxxx
let add_one (x:int) : int  = 1 + x

Like C, you have to declare a function before you use it, not after.

Remember, everything to left of colon is a value, and to the immediate right is a value. So, you can leave off types with:

xxxxxxxxxx
let add_one x = 1 + x

Local function definition occurs w/in another function:

xxxxxxxxxx
let add_two (x:int) : int = 
  let add_one x = 1 + x in
  add_one (add_one x)

Function with two arguments looks like this:

xxxxxxxxxx
let add (x:int) (y:int) : int = x + y

The type of add_one is int -> int, the type of add is int -> int -> int.

Type Checking a Function:

For add, you can call add (3+4) 7 which is 100% legal.

For a function with type A -> B -> C -> D -> ..., everytime you add an argument, you can get the type of the result by knocking off one of the letters from the left.

You can never return two different types from a function.

Tuples

A tuple is a fixed, finite, ordered collection of values.

To use a tuple, we extract its components, .

xxxxxxxxxx
(* let (id1, id2, ..., idn) = e1 in e2 *)
let (x,y) = (2,4) in x + x + y (* x = 2, y  = 4, use in expr. *)

Steps:

  1. Evaluate thing to the right of the equal sign.
  2. Pluck out components and substitute in to the identifiers on the left of equal sign.
  3. Evaluate expression to the right of the in keyword.

Rules for Typing Tuples: if e1:t1 and e2:t2 then (e1,e2) : t1 * t2. Overall expression takes on the type of e2.

Problem Solving Steps:

  1. Write down the function and argument names.
  2. Write down argument and result types.
  3. Write down some examples (in a comment).
  4. Deconstruct input data structures.
  5. Build new output values.
  6. Clean up by identifying repeated patterns (potentially with helper functions).

Actually, the unit type is just a tuple with 0 fields: () : unit.

Let Expressions

Write code like a mathematical proof; instead of (2 + 3) * (2 + 3), we write let x = 2 + 3 in x * x.

We can have a series of let expressions by just using the in keyword (we call this cascading or nested).

.ml File Structure: Every .ml file is a sequence of declarations (a little different than raw expressions). Declarations do not end with in, just of the form let <var> = <expr>. Within declarations, you can have nested expressions.

OCaml variables are immutable. Once bound to a value, a variable is never modified or changed. When in question, work outwards and upwards through a program to find the closest enclosing definition. That is the value of this use forever and always. If you explicitly assign a different value with a new let statement, you create a new variable to be used from there on in the code. If a function calls a variable and you use the function after reassigning the variable, it still doesn't change the function.

For nested let expressions always evaluate to the right of the nearest equal sign then just substitute the values in the expression following the in keyword and repeat.

let expressions really are expressions, just larger ones.

Type Checking

When explaining the OCaml type checking rules, we write exp : tto mean "the expression exp is well typed and has type t."

Example "proof":

xxxxxxxxxx
because   bool_of_string : string -> bool
and               "true" : string
we conclude that (bool_of_string "true") : bool

The type for get is string -> int -> char and since function types are right associative, this type is equivalent to string -> (int -> char).

Note we typically write

xxxxxxxxxx
(get "Dave") 3

as

xxxxxxxxxx
get "Dave" 3

We may do this because functional application is left associative. The expressions with and without parentheses are equivalent.
